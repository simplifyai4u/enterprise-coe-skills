# API Design and Governance

Use this skill when shaping APIs, standards, or developer-facing interfaces.

## Objectives
- Define API contracts and standards
- Improve usability, security, and versioning
- Support governance and lifecycle management

## Instructions
1. Clarify consumer needs and API purpose.
2. Define endpoints, payloads, authentication, and error handling.
3. Recommend versioning, documentation, and lifecycle practices.
4. Summarize governance controls and rollout guidance.
