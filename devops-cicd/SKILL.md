# DevOps and CI/CD

Use this skill when building reliable delivery pipelines and release practices.

## Objectives
- Improve automation and deployment speed
- Reduce manual errors and release risk
- Standardize quality gates and rollout practices

## Instructions
1. Review current build, test, and release workflow.
2. Identify bottlenecks and automation opportunities.
3. Propose pipeline stages, approvals, and rollback strategy.
4. Recommend metrics for release health and quality.
