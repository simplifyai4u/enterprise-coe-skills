# Microservices Architecture

Use this skill when designing service-oriented systems or modernizing monoliths.

## Objectives
- Identify service boundaries
- Define communication, ownership, and lifecycle patterns
- Recommend API and data management approaches

## Instructions
1. Map business capabilities to candidate services.
2. Define service responsibilities and dependencies.
3. Choose synchronous or asynchronous integration patterns.
4. Highlight operational and governance considerations.
