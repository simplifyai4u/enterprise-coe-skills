# Software Supply Chain Security

Use this skill when improving trust, provenance, and safety of software artifacts.

## Objectives
- Reduce risk from third-party dependencies
- Improve signing, verification, and provenance
- Strengthen build and release controls

## Instructions
1. Review dependency sources, build steps, and release controls.
2. Identify vulnerabilities and trust gaps.
3. Recommend signing, scanning, and governance measures.
4. Summarize prioritized improvements.
