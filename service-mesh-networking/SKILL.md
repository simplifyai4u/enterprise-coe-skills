# Service Mesh and Networking

Use this skill when improving internal service connectivity, resilience, and traffic management.

## Objectives
- Define service discovery and traffic routing patterns
- Improve reliability and security for service communication
- Support observability and policy enforcement

## Instructions
1. Review service communication needs and constraints.
2. Recommend mesh or network patterns and control planes.
3. Define traffic policies, security controls, and failover behavior.
4. Summarize operational considerations.
