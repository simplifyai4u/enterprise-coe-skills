# Observability and Monitoring

Use this skill when improving visibility into application and platform health.

## Objectives
- Define key metrics, logs, and traces
- Establish alerting and dashboards
- Support root cause analysis and incident readiness

## Instructions
1. Identify critical system signals and user-impacting metrics.
2. Recommend telemetry collection strategies.
3. Define alert thresholds and response routing.
4. Summarize dashboard and incident workflow guidance.
