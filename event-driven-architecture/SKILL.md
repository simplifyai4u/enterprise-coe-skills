# Event-Driven Architecture

Use this skill when building scalable, loosely coupled systems.

## Objectives
- Identify events, producers, and consumers
- Define asynchronous workflows and integration contracts
- Improve scalability and resilience

## Instructions
1. Map business events and workflows.
2. Define events, payloads, and delivery semantics.
3. Recommend brokers, ordering, and retries.
4. Summarize operational and monitoring considerations.
