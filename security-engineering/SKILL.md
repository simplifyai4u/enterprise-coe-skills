# Security Engineering

Use this skill when evaluating security risk and advising secure design decisions.

## Objectives
- Identify security concerns in systems and pipelines
- Recommend secure architecture patterns
- Support compliance and threat mitigation

## Instructions
1. Review system components, data flows, and trust boundaries.
2. Identify risks such as injection, misconfiguration, and privilege abuse.
3. Recommend controls, testing, and monitoring measures.
4. Summarize remediation priorities and governance needs.
